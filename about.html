태스크 기반의 디자인 가이드로 서비스 초기 구축 효율화하기


본 연구는 앱 시장의 성숙에 따라 소셜 네트워킹이나 쇼핑 등 서비스의 이용 목적 별로 서비스의 인터페이스가 유사성을 띠고 있다는 점에 착안하여, 비전문가나 디자이너가 빠르고 효율적으로 초기 서비스 구조를 디자인할 수 있도록 가이드를 만드는 데 그 목적이 있다.

기존 인터페이스 디자인 가이드가 컴포넌트 단위를 구체적으로 설명한다면, 본 가이드는 서비스의 이용 목적에 따른 태스크(task, 과업)로 필요한 화면과 컴포넌트를 설명한다. 가이드 제작을 위해 앱 서비스 스크린샷 라이브러리 웹사이트인 Pttrns를 스크래핑하여 인터페이스 특징과 앱 마켓 카테고리 데이터를 추출하여 분석하였다.

본 가이드의 활용 방법은 다음과 같다.

01
서비스 사용 목적 정의

제작하고자 하는 서비스의 사용 목적을 정의한다.

Example

지인 추천 기반의 맛집 예약 서비스

02
사용 목적에 따른 기능 정의

사용 목적 달성에 필요한 주요 및 부가 기능을 정의한다.

Example

  • 지인 추천
    • 지인의 추가
    • 신규 지인의 서비스 초대
    • 맛집 업로드
  • 맛집 큐레이션
  • 맛집 예약
  • 사용자 및 지인 활동 알림
  • 장소에 대한 방문 기록
03
기능을 태스크로 맵핑

앞서 정의한 기능을 본 가이드의 태스크로 맵핑한다.

Example

  • 지인 추천
    • 지인의 추가 : Networking
    • 신규 지인의 서비스 초대 : Invite Friends
    • 맛집 업로드 : Create Post
  • 맛집 큐레이션 : News Feed
  • 맛집 예약 : Purchase & Reservation
  • 사용자 및 지인 활동 알림 : Activity Feeds
  • 장소에 대한 방문 기록 : Check In
04
태스크에 정의된 화면 흐름 연결

각 태스크의 화면 흐름을 연결하여 사용자의 유기적인 흐름을 완성한다. 유기적인 사용 흐름을 위해 연결이 단절된 플로우는 필요한 부가 태스크를 추가add 하여 연결한다.


05
‘Base’ 태스크 화면 플로우 연결

모든 서비스에 공통적으로 나타나는 화면으로, 앞서 정의한 화면 플로우 간 연결에 서비스 제작의 기초가 되는 ‘Base’ 태스크의 화면 플로우를 연결한다.

이때 중요한 작업은 홈 화면의 정의이다. 홈 화면은 서비스 구동 초기에 사용자가 이용 목적에 부합하는 정보를 직관적으로 발견하고 다른 화면으로 이용 흐름이 연결될 수 있도록 하는 중요한 영역이다.

본 서비스의 경우 ‘맛집 발견’, 즉 ‘News Feed’ 태스크의 화면을 홈 화면으로 정의 settings_ethernet 하여 다른 태스크로의 유기적인 연결을 유도한다.

Example

  1. Networking → Contentsadd : 사용자 프로필 페이지에서 해당 사용자가 작성한 콘텐츠 선택 시 해당 콘텐츠로 이동할 수 있어야 한다. 따라서 ‘Content’ 태스크가 추가되었다.
  2. Contents → Purchase & Reservation : 콘텐츠 페이지에서 맛집 정보 선택 시 해당 맛집 정보를 상세히 살펴볼 수 있도록 ‘Purchase & Reservation’ 태스크의 제품 페이지(여기에서는 레스토랑 정보 페이지)로 이동할 수 있도록 연결한다.
  3. News Feed → Networking : 뉴스피드의 사용자 정보를 선택하면 사용자 프로필 페이지로 이동할 수 있도록 연결한다.
  4. News Feed → Contents : 뉴스피드의 프리뷰 콘텐츠를 선택하면 콘텐츠 페이지로 이동할 수 있도록 연결한다.
  5. News Feed → Searchadd : 뉴스피드의 검색 버튼 선택 시 원하는 콘텐츠를 검색할 수 있도록 ‘Search’ 태스크가 추가되었다.
  6. News Feed → Searchadd : 뉴스피드에서 원하는 콘텐츠만 탐색할 수 있도록 필터 기능을 추가한다.
  7. Navigation Menuadd → Create Post : 맛집 탐색 외 맛집 업로드와 같이 서비스가 제공하는 기능들을 병렬 노출하기 위해 ‘Navigation Menu’ 를 추가하고, 메뉴 중 맛집 업로드 기능을 선택하면 ‘Create Post’ 태스크로 연결하여 업로드 할 맛집의 정보를 작성할 수 있도록 한다.
  8. Create Post → Check In : 뉴스피드의 사용자 정보를 선택하면 사용자 프로필 페이지로 이동할 수 있도록 연결한다.
  9. Navigation Menu → Activity Feeds : 맛집 탐색 외 맛집 업로드와 같이 서비스가 제공하는 기능들을 병렬 노출하기 위해 ‘Navigation Menu’ 를 추가하고, 메뉴 중 맛집 업로드 기능을 선택하면 ‘Create Post’ 태스크로 연결하여 업로드 할 맛집의 정보를 작성할 수 있도록 한다.
  10. Navigation Menu → News Feed : 서비스 메뉴에서 뉴스피드 메뉴를 선택하여 최근 콘텐츠 목록을 탐색할 수 있도록 한다.
  11. Baseadd → Invite Friends : 리워드 제공으로 지인 추천을 촉진하는 ‘Invite Friends’ 태스크는 ‘Create Post’ 태스크와 마찬가지로 세부 기능이나, 서비스 제공자가 전략적으로 판단하여 ‘Navigation Menu’ 포함 여부를 결정할 수 있다. 마이너 기능으로 제공할 경우 ‘Base’ 태스크의 설정 화면 내 기능 목록 중 하나로 연결하여 사용자가 발견할 수 있도록 한다.
본 가이드의 한계 및 후속 연구 사항

연구자는 본 연구를 통해 웹과 앱 서비스의 인터페이스가 사용 목적 별로 구조적 패턴이 있음을 발견할 수 있었다. 그리고 다음을 본 연구의 한계 및 후속 연구 과제로 제시한다.

사용 목적 별 인터페이스 특징을 조사하고자 수집된 데이터는 데이터 수집처의 제작자 의도가 반영된 것이다. 따라서 본 연구에 포함된 서비스라 하더라도 해당 서비스의 전체 페이지의 데이터가 수집되지 않았으므로 본 연구가 발견하지 못한 태스크나 추가 해석 사항이 존재할 수 있다.

본 연구는 디자이너 뿐만 아니라 비전문가의 서비스 제작 편의를 도모하기 위한 것이었다. 하지만 서비스 제작자가 의도하는 사용 목적 별로 태스크를 맵핑하는 것, 그리고 태스크 간 화면을 연결하는 작업은 사용자 경험 디자인 영역에 대한 일정 수준의 이해를 필요로 한다. 따라서 각 태스크를 직관적으로 이해하여 서비스 제작에 사용할 수 있도록 적합한 태스크명을 재정의하고, 비전문가도 태스크 간 화면을 유기적으로 연결시킬 수 있도록 화면 내 컴포넌트에 대한 상세한 가이드가 필요하다.

서비스를 구성하는 화면들이 유기적으로 연결되어 있다는 것이 곧 좋은 사용성을 보장하지 않는다. 사용자가 각 화면에서 정보를 보다 잘 해석하고 필요한 기능을 발견할 수 있도록 시지각 이론에 기반한 컴포넌트 디자인 연구 또한 필요하다.

서비스가 추구하는 가치를 실현할 수 있는 사용자가 얼마나 많은지, 그리고 얼마나 길게 혹은 자주 서비스에 머무르는지가 곧 서비스 완성도의 척도이다. 따라서 사용자의 사용 목적 달성과 서비스 이탈 방지를 위한 행동 데이터 분석 방법과 그에 따른 서비스 디자인 개선 방법 또한 본 연구의 후속 과제라 할 수 있다.
