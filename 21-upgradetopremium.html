Upgrade to Premium


구독 유도, 유료 버전 전환

‘Upgrade to Premium’ 태스크는 OTT(over-the-top media service), 음악, 뉴스 서비스와 같이 서비스 제작자가 제공하는 콘텐츠 이용을 위해 과금 플랜을 확인하고 결제하는 구독 형태와 메모, 일정 관리, 계산기 등 특정 작업을 완수할 수 있는 기능을 제공하는 서비스에서의 유료 버전 전환 형태로 구분된다.

구독 트리거 (Subscription trigger)

사용자에게 구독 유인을 제공하기 위해 일반적으로 구독 설정 전에도 서비스가 제공하는 콘텐츠 목록과 해당 콘텐츠의 프리뷰를 살펴볼 수 있도록 제공한다. 프리뷰 이후에도 콘텐츠를 지속 이용하고자 할 때 서비스의 과금 플랜을 확인하고 선택할 수 있도록 구독 트리거를 노출한다. OTT 서비스나 음악 서비스와 같이 재생이 주된 서비스 행위인 경우, 재생 버튼 선택 시 혹은 미리보기 콘텐츠 재생 후 시스템 컨펌(Confirmations) 노출로 구독 전환을 유도한다. 뉴스 서비스와 같은 읽기 콘텐츠는 프리뷰 콘텐츠 말미에 구독 버튼을 노출하여 전환을 유도한다.

유료 버전 전환 트리거 (Upgrade trigger)

특정 작업에 대한 서비스가 무료 버전일 경우 기능이 제한적이거나 광고가 노출되며, 사용자는 심화 기능 이용이나 광고 없이 서비스를 이용하기 위해 유료 버전 전환을 선택한다. 트리거는 기능의 제한을 해제하려 하거나 광고 배너 영역에 노출하여 유료 버전 전환을 유도한다.

과금 플랜 확인 및 결제 방법 등록 (Pricing plans)

트리거 선택 후 사용자가 본인에게 적합한 과금 플랜을 선택하여 결제 프로세스로 연결할 수 있도록 해당 서비스가 제공하는 과금 플랜을 안내한다. 구독이나 유료 전환 기반의 서비스는 쇼핑 서비스와는 다르게 단일화된 결제 방법을 제공한다. 일반적으로 결제할 카드 정보를 등록할 수 있도록 하며, 카드 정보 입력 절차의 간소화 혹은 현실적인 개발 여건 등으로 애플리케이션 마켓을 통해 구독 서비스로 등록할 수 있도록 유도하기도 한다.

결제 확인 (Confirm payment)

구독 및 유료 전환 완료 확인 화면으로 구독이나 유료 전환 유효 및 갱신 기간과 이로인해 사용자가 얻게된 유익을 안내한다. 구독 서비스에서는 구독 완료 메시지가 노출된 시스템 컨펌창을 노출하고, 사용자가 유료로 이용하고자 했던 콘텐츠로 연결하여 해당 콘텐츠를 이용 흐름이 원활하도록 한다.
